# Pytest 実行結果の不具合分析とレビューレポート

**作成日:** 2026年3月24日
**ステータス:** 要対応 (Critical)
**サマリー:** 28件中17件のエラー（セットアップ失敗）、7件の失敗が発生。主にDB環境の不備とモジュール設計の不整合が原因。

---

## 1. シニアエンジニアの視点：アーキテクチャと設計の整合性
**指摘事項：インターフェースの乖離とモジュール境界の崩れ**

*   **`utils.api` モジュールの不整合:**
    *   `AttributeError` が多発しており、テストが参照している `standardize_raw_data`, `load_config`, `_financial_item_mapping` 等が `utils/api.py` に存在しない。
    *   リファクタリングによってロジックが `data_mapper.py` や `parser.py` に移動した可能性が高いが、テストコードが古いインターフェースに依存したままになっている。
*   **名前空間の管理:**
    *   プライベート関数（`_`で始まるもの）を外部から直接テストしている箇所があり、テストの結合度が高すぎる。

**修正アクション:**
- `utils/api.py` のエクスポート内容を現状のコードベースと同期させる。
- ロジックが移動している場合は、`tests/test_mapping.py` のインポート先を適切なモジュールへ修正する。

---

## 2. CIエンジニアの視点：実行環境とライフサイクル管理
**指摘事項：データベースのプロビジョニング不全**

*   **テーブル未定義エラー (`UndefinedTable`):**
    *   `relation "companies" does not exist` のエラーは、CI環境のPostgreSQLに対してDDL（テーブル作成）が実行されていないことを示している。
*   **テスト用DBの初期化フローの欠落:**
    *   `db_session` フィクスチャ内で `delete()` を呼ぶ前に、`Base.metadata.create_all(engine)` によるテーブル作成が行われていない。
*   **環境変数の整合性:**
    *   CI上の `localhost:5432` への接続設定が、テストランナーから正しく到達できているか、また権限が適切か再確認が必要。

**修正アクション:**
- `tests/conftest.py` 等で、テストセッション開始時に必ず `create_all` を実行し、終了時にクリーンアップする処理を実装する。
- GitHub Actions のワークフローで、DBのヘルスチェック完了後にテストを開始するように設定を強化する。

---

## 3. QAエンジニアの視点：テストの信頼性と品質保証
**指摘事項：テストの形骸化とリグレッション検出能力の喪失**

*   **広範囲なセットアップ失敗による検証不能:**
    *   エラーの大半がテスト実行前のセットアップ段階（フィクスチャ）で発生しており、本来のビジネスロジックの検証まで到達できていない。
*   **外部ファイル依存の脆弱性:**
    *   `documents/test.csv` の読み込みに失敗しており、テストデータのポータビリティが低い。
*   **モック戦略の不備:**
    *   `api.config` 等のグローバルな状態に対する `patch.dict` が失敗しており、テストの隔離が不十分。

**修正アクション:**
- **Redの解消:** まず `AttributeError` を解消し、テストが「失敗（Fail）」を報告できる状態まで引き上げる。
- **インメモリDBの検討:** リポジトリテストにおいて、外部DB依存を減らすために SQLite（`:memory:`）の利用を検討する。
- **フィクスチャの堅牢化:** ファイル依存を減らし、コード内で完結する最小限のテストデータ（Data Factory 等）を利用する。

---

## 4. 優先修正ポイントのまとめ

1.  **[最優先] DBセットアップの修正:** `UndefinedTable` を解消するため、テストコード内でDDLを実行するように修正。
2.  **[重要] API定義の同期:** `utils/api.py` の現状に合わせてテストのインポートと呼び出しを修正。
3.  **[恒久対策] CI環境の同期:** `uv sync` やマイグレーションの実行をCIパイプラインに正しく組み込む。
